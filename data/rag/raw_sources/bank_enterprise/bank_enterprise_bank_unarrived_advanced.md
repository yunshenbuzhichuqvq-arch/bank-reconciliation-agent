# 银企对账银行未到账进阶场景（演示）

> 本文件为演示规则，非真实银行内部制度。判定依据仅标注公开制度方向，供 RAG 检索演示使用。

## [BE-R005] 票据或汇票未兑付的未到账

- 适用场景：企业已按票据确认收款，但票据尚未托收兑付，银行侧无入账，形成单边未到账。
- 判定条件：present_side == A（企业已记、银行未达）；存在票据/汇票凭证但无兑付流水，未超提示付款期。
- 处理建议：按票据状态追踪托收进度，未兑付前不冲销；到期未兑付提示退票风险并转人工。
- 判定依据：参考方向票据托收与提示付款期管理方向（公开制度方向，非内部制度）。
- 示例：企业 6/10 收银行承兑汇票并挂账，托收中银行尚未入账 → 银行未到账，跟踪兑付。

本项目抽象规则：`BANK_UNARRIVED`（BE-R005）涵盖票据未兑付的单边未达，按票据状态跟踪，未兑付不得冲销。

## [BE-R005] 跨行或跨境清算延迟的未到账

- 适用场景：跨行大额或跨境汇款因清算链路较长，企业已记而银行尚未入账。
- 判定条件：present_side == A；交易经跨行/跨境清算，处于正常清算时限内，尚无到账流水。
- 处理建议：在清算时限内保持挂账并跟踪；超时限仍未到账则发起查询查复并转人工。
- 判定依据：参考方向跨行/跨境清算时限与查询查复方向（公开制度方向，非内部制度）。
- 示例：跨境汇款 T+2 清算，企业已记、银行 T+1 尚无入账 → 时限内未到账，继续跟踪。

本项目抽象规则：跨行/跨境清算延迟属 BE-R005 的正常未达，须以清算时限判断，超时未达升级查询查复。

## [BE-R005] 与跨期入账的区分（干扰：vs CROSS_PERIOD_POSTING）

- 适用场景：判断"银行始终尚无该笔"（未到账）还是"银行已在相邻期间入账"（跨期）。
- 判定条件：未到账=银行侧 anti-join 始终无该笔；跨期=银行侧在相邻期间已能查到、仅记账期间不同。
- 处理建议：先 anti-join 确认银行侧是否最终存在该笔；不存在→BE-R005；存在但跨期→CROSS_PERIOD_POSTING。
- 判定依据：参考方向未达账项与在途资金分类方向（公开制度方向，非内部制度）。
- 示例：企业 6/29 记账，银行至今无该笔 → 未到账；银行 7/1 已入同笔 → 跨期。

本项目抽象规则：银行未到账与跨期入账以"银行侧是否最终存在该笔"切分；始终不存在=BE-R005，存在仅跨期=CROSS_PERIOD_POSTING。

## [BE-R005] 长期挂账未达的预警与升级

- 适用场景：未达项长期未闭环，存在虚挂收入或资金风险，需预警升级。
- 判定条件：present_side == A 且挂账天数超过设定预警阈值（如 > 30 天）仍无到账或查复结果。
- 处理建议：触发预警并升级人工/管理复核，核查交易真实性与对手履约；不得长期无依据挂账。
- 判定依据：参考方向往来款项账龄分析与挂账清理方向（公开制度方向，非内部制度）。
- 示例：某未达项挂账 45 天无到账、无查复 → 预警升级核查真实性。

本项目抽象规则：BE-R005 长期未达须按账龄预警并升级核查，避免虚挂；超阈值无依据的挂账须转人工。

## [BE-R005] 银行未到账转人工的触发条件（处理规范）

- 适用场景：明确未达项何时必须转人工，何时可继续 Agent 跟踪。
- 判定条件：超清算时限/票据到期未兑付/超账龄预警/查询查复无果/RAG 无命中，任一满足即升级。
- 处理建议：升级 PENDING_HUMAN 并附票据状态、清算时限、账龄与查复记录；RAG 无命中直接转人工不触发 Fallback。
- 判定依据：参考方向无依据不判定与人工兜底方向（公开制度方向，非内部制度）。
- 示例：查询查复无果且超账龄 → 转人工并附全部跟踪记录。

本项目抽象规则：BE-R005 在超时限/超账龄/查复无果/RAG 无命中时必须转人工，证据链须含票据与清算跟踪记录。
