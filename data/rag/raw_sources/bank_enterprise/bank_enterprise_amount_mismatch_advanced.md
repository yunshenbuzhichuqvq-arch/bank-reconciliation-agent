# 银企对账金额差异进阶与边界场景（演示）

> 本文件为演示规则，非真实银行内部制度。判定依据仅标注公开制度方向，供 RAG 检索演示使用。

## [BE-R002] 大额金额差异的分级复核阈值

- 适用场景：金额差异按绝对值与占比分级，控制大额差错的复核优先级与审批层级。
- 判定条件：flow_matched == true 且 amount_diff != 0；差额绝对值或占交易金额比例超过设定阈值（如 ≥ ¥10,000 或 ≥ 5%）触发优先复核。
- 处理建议：高于阈值→优先进入 AuditAgent 并提升审批层级；低于阈值→常规复核。阈值不替代取证，仍需双边金额。
- 判定依据：参考方向重要性水平与分级授权审批的内部控制方向（公开制度方向，非内部制度）。
- 示例：交易 ¥500,000、差额 ¥40,000（8%）超阈值 → 优先复核并上提审批。

本项目抽象规则：金额差异分级仅决定复核优先级与审批层级，不改变 `AMOUNT_MISMATCH`（BE-R002）须带双边金额的取证要求。

## [BE-R002] 退款或冲正与原交易抵消后的净额差

- 适用场景：原交易后发生退款/冲正，企业与银行对"净额"的体现节奏不同，造成阶段性金额差。
- 判定条件：存在原交易与退款/冲正成对记录，但一侧已抵消、另一侧尚未，导致净额暂不相等。
- 处理建议：成对核对原交易与退款，按净额口径解释差异；未抵消侧提示补登抵消分录。
- 判定依据：参考方向红冲/退款与净额列报核对方向（公开制度方向，非内部制度）。
- 示例：原收款 ¥6,000、退款 ¥6,000，企业已净额、银行尚有两笔未轧差 → 按净额解释。

本项目抽象规则：退款/冲正净额差属 BE-R002 的阶段性差异，须成对核对原交易与退款，警惕与重复记账（BE-R008）混淆。

## [BE-R002] 多发票合并收款的金额归集差异

- 适用场景：客户一笔付款覆盖多张发票，企业按发票分别挂账，合并与拆分口径不同造成差异。
- 判定条件：银行单笔金额=企业多张发票应收之和；逐张挂账与单笔到账口径不一致。
- 处理建议：以付款单关联多张发票核对总额；总额一致则按归集解释，缺口按差异挂账。
- 判定依据：参考方向应收账款核销与发票匹配核对方向（公开制度方向，非内部制度）。
- 示例：银行到账 ¥9,000 对应三张发票 ¥3,000×3 → 归集核对一致。

本项目抽象规则：合并收款差异以"付款总额与多发票之和是否一致"判定，闭合则归集解释，差额仍归 BE-R002。

## [BE-R002] 外币汇率重估造成的账面差异

- 适用场景：外币往来在资产负债表日按期末汇率重估，账面本币额与银行入账本币额产生差异。
- 判定条件：原币一致但本币不等，差异由汇率重估（期末汇率 vs 入账汇率）造成，区别于舍入尾差。
- 处理建议：核对原币、入账汇率与期末汇率，按汇兑损益处理重估差异，不得当作本金差错。
- 判定依据：参考方向外币交易期末折算与汇兑损益核算方向（公开制度方向，非内部制度）。
- 示例：USD 10,000，入账 7.10、期末 7.20，本币差 ¥1,000 由重估产生 → 计汇兑损益。

本项目抽象规则：汇率重估差异属 BE-R002 的特殊情形，处理方向是确认汇兑损益，与舍入尾差、本金错误区分。

## [BE-R002] 与手续费或税费差异的边界区分（干扰：vs BE-R003）

- 适用场景：差额无法由费用/税费解释时，应留在金额不一致，而非误归费税差异。
- 判定条件：amount_diff 不命中手续费特征集、不等于显式手续费字段、无法对应任何税率。
- 处理建议：保留双边金额与差额，提示检查方向、拆分、退款与本金错误；不强行套用费税解释。
- 判定依据：参考方向金额差异保留双边口径的核对方向（公开制度方向，非内部制度）。
- 示例：差额 ¥712.40 无费率/税率对应 → 留在 BE-R002，不转 BE-R003。

本项目抽象规则：金额不一致与费税差异以"差额能否被手续费/税费解释"切分，不可解释者归 `AMOUNT_MISMATCH`（BE-R002）。

## [BE-R002] 金额差异升级人工复核的触发条件（处理规范）

- 适用场景：明确哪些金额差异不应由 Agent 自动结论，必须升级人工。
- 判定条件：差额无可解释来源、证据缺失双边金额、或差异涉及大额/异常方向且 RAG 无命中。
- 处理建议：满足任一升级条件即 PENDING_HUMAN，并附已采集字段与缺失项清单；RAG 无命中时不触发 Fallback，直接转人工。
- 判定依据：参考方向无依据不判定与人工复核兜底方向（公开制度方向，非内部制度）。
- 示例：差额无来源且 RAG 未命中规则 → 直接转人工，记录缺失证据。

本项目抽象规则：BE-R002 在"无解释来源/证据不足/RAG 无命中"时必须升级人工，禁止凭低依据自动平账。
