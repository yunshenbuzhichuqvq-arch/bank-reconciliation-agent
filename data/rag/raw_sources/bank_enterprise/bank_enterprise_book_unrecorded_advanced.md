# 银企对账企业未入账进阶场景（演示）

> 本文件为演示规则，非真实银行内部制度。判定依据仅标注公开制度方向，供 RAG 检索演示使用。

## [BE-R006] 银行自动扣费（账户管理费/短信费）企业漏记

- 适用场景：银行按月自动扣收账户管理费、短信通知费等小额费用，企业未及时入账，形成单边。
- 判定条件：present_side == B（银行已扣、企业未记）；银行流水为小额规律性扣费，企业账簿无对应分录。
- 处理建议：归集银行自动扣费明细，提示企业按费用科目补记；区别于双边都存在的费用差异。
- 判定依据：参考方向银行服务收费与费用据实入账方向（公开制度方向，非内部制度）。
- 示例：银行每月扣账户管理费 ¥10.00，企业连续三月未记 → 企业未入账，补记费用。

本项目抽象规则：`BOOK_UNRECORDED`（BE-R006）涵盖银行自动扣费的单边未记；这是企业完全漏记该笔，区别于双边的手续费差异（BE-R003）。

## [BE-R006] 利息计提银行已入企业未记（注意与费用差异区分）

- 适用场景：银行结息/计提利息已入账，企业未登记利息收入或利息支出，形成单边未记。
- 判定条件：present_side == B；银行有结息流水，企业无对应利息分录（而非双边都记仅金额差）。
- 处理建议：按利息收入/支出科目补记；若双边都已记仅金额因税费不同，则应走 BE-R003 而非本规则。
- 判定依据：参考方向利息确认与结息入账方向（公开制度方向，非内部制度）。
- 示例：季度结息 ¥320.00 银行已入、企业未记 → BE-R006 补记利息收入。

本项目抽象规则：利息单边未记归 BE-R006；若双边都记而差额来自利息税费，则归 `FEE_TAX_DIFF`（BE-R003），二者以"企业是否记了该笔"切分。

## [BE-R006] 代收款项银行已入企业未确认

- 适用场景：银行代收货款/代收费已入企业账户，但企业因未收到通知或对账滞后未确认入账。
- 判定条件：present_side == B；银行有代收入账流水，企业无收入确认分录。
- 处理建议：核对代收协议与到账明细，提示企业确认收入并补记；保留代收回单。
- 判定依据：参考方向委托代收与收入确认核对方向（公开制度方向，非内部制度）。
- 示例：银行代收货款 ¥18,000 已入账，企业未确认 → 补记应收核销与收入。

本项目抽象规则：代收款项单边未记归 BE-R006，处理方向是确认收入并补记，留存代收回单为证据。

## [BE-R006] 与银行未到账的方向区分（干扰：vs BE-R005）

- 适用场景：单边残留须先判方向，避免把"企业未记"与"银行未到"搞反，导致处理路径错误。
- 判定条件：BE-R006=银行有、企业无（present_side == B）；BE-R005=企业有、银行无（present_side == A）。
- 处理建议：以 present_side 确定方向；B 侧单边→企业补记（BE-R006），A 侧单边→跟踪银行到账（BE-R005）。
- 判定依据：参考方向未达账项按方向分类编表方向（公开制度方向，非内部制度）。
- 示例：银行有该笔企业无 → BE-R006；企业有该笔银行无 → BE-R005。

本项目抽象规则：企业未入账与银行未到账以 present_side 方向严格切分（B vs A），处理路径相反，不可混淆。

## [BE-R006] 企业未入账的差错升级（处理规范）

- 适用场景：企业未入账涉及金额较大、时间较久或科目不明时的升级处理。
- 判定条件：未记金额超阈值、跨多个期间、科目归属不明或 RAG 无命中规则。
- 处理建议：升级人工确认科目与归属期间后补记；RAG 无命中直接转人工，不触发 Fallback。
- 判定依据：参考方向重大差错更正与人工复核方向（公开制度方向，非内部制度）。
- 示例：一笔 ¥120,000 代收款科目不明且跨期 → 转人工确认后补记。

本项目抽象规则：BE-R006 在大额/跨期/科目不明/RAG 无命中时升级人工，确认科目与归属期后再补记。
