# 清算文件异常处理指引（模拟摘录）

## 清算文件缺失

清算文件未到达时，应先核对文件传输状态、渠道通知、批次日期和重传窗口。不得用上一批次文件代替本批次清算依据。

本项目抽象规则：文件缺失只能输出待补文件或人工追踪动作，不能生成自动平账建议。

## 文件重复导入

同一清算批次文件重复导入会造成核心侧或清算侧虚增流水。复核时应比较文件名、批次号、渠道机构号、记录总数和总金额。

本项目抽象规则：重复导入证据闭环时应提示撤销重复批次或隔离重复数据，并记录重复文件标识。

## 文件格式或字段缺损

文件格式异常、关键字段为空或编码错误时，应先拒绝自动对账，保留原始文件、解析错误和缺失字段清单。

本项目抽象规则：缺少交易参考号、金额或交易时间时，TraceAgent 不得构造候选匹配。

## 文件日期与清算日期不一致

文件名日期、文件内清算日期和业务交易日期不一致时，应以渠道清算规则确认归属批次，并检查是否为跨日切或补发文件。

本项目抽象规则：日期口径冲突时应输出需人工确认的日期字段，不得默认采用系统导入日期。
