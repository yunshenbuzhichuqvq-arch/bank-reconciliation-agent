# ADR-001: MVP-1 ExceptionRouter 采用 §15.2 纯银企五分支

- Status: Accepted (2026-06-05)
- Deciders: 用户（拍板）, Planner agent（提案）
- Related: system-prd.md §3.2 / §15.2, overall-architecture.md §5.1, TASK-103, TASK-104, TASK-105, TASK-108

## Context

PRD 内部对 "MVP-1 ExceptionRouter 第一版覆盖哪 5 个核心分支" 有两处互相冲突的表述：

- §3.2（新增清单）：`金额差错 / 单边缺失 / 跨日切单边 / 模糊冲正 / 疑似重复入账`
- §15.2（验收标准）：`金额不一致 / 银行未到账 / 企业未入账 / 摘要/客户名不一致 / 疑似重复记账`

差异点：§3.2 含 `跨日切单边`、`模糊冲正` 两个偏**清算对账**语义的分支；§15.2 全部是**银企对账**分支。
而 MVP-1 的定位是「银企对账主链路、本地可演示」，双场景（银企 + 清算）要到 MVP-2a 才正式引入。

架构 overall-architecture.md §5.1 已给出银企对账分支的规范定义与 ID：BE-R001..R008。

## Options

1. **以 §15.2 验收为准的纯银企五分支**（对齐架构 §5.1）。
2. 以 §3.2 为准，提前引入 `跨日切 / 模糊冲正` 等清算语义分支。
3. 自定义其它 5 分支组合。

## Decision

采用 **Option 1**。MVP-1 ExceptionRouter 第一版覆盖以下 5 个银企对账异常分支（外加 BE-R001 精确匹配自动平账）：

| 分支 ID | error_type | exception_branch | 语义 |
|--------|-----------|------------------|------|
| BE-R002 | `AMOUNT_MISMATCH` | `BE-R002` | 流水可匹配但金额不一致 |
| BE-R004 | `NARRATIVE_NAME_MISMATCH` | `BE-R004` | 金额一致但摘要/客户名不一致（含模糊冲正关键词的**确定性**识别） |
| BE-R005 | `BANK_UNARRIVED` | `BE-R005` | 企业账簿(A)有、银行(B)无 |
| BE-R006 | `BOOK_UNRECORDED` | `BE-R006` | 银行(B)有、企业(A)无 |
| BE-R008 | `DUPLICATE_BOOKING` | `BE-R008` | 同主体+同金额+同对手，疑似一端多记 |

附 BE-R001 `EXACT_MATCH` → 自动平账（非异常分支，但属规则引擎覆盖）。

落地约定：
- `跨日切`（CROSS_PERIOD_POSTING / 清算日切）推迟到 MVP-2a 清算副链路。
- `模糊冲正` 不单列分支，并入 BE-R004，在 NARRATIVE 分支里做**确定性关键词识别**（冲正/红冲/退款/抹账），不依赖 LLM。
- **A/B ↔ 现有表槽位映射**（外科手术式、不改列模型，见 ADR-002）：
  - Source B（银行流水）= 现有 `bank_*` 槽位；Source A（企业账簿）= 现有 `clear_*` 槽位。
  - 因此：**仅 bank 侧存在** → `BOOK_UNRECORDED`（银行有/企业无）；**仅 clear 侧存在** → `BANK_UNARRIVED`（企业有/银行无）。

## Consequences

- §3.2 与 §15.2 的措辞冲突由 Planner 同步修正（仅改文档），统一到本 ADR 的五分支。
- MVP-1 不出现任何清算专有分支；mock 数据（TASK-108）按这 5 分支构造。
- MVP-2a 引入双场景时，ExceptionRouter 升级为银企全集 + 清算分支，本 ADR 的五分支是其子集，不会返工。
