这是我转行求职的个人开源项目。目标岗位是 Agent 开发工程师 / AI 应用开发工程师。
现有的 overall-architecture.md 和 system-prd.md 对一个个人项目来说严重过度工程化了
——更像一个"全栈 + 运维"项目,稀释了我真正想展示的 Agent 工程信号。
项目目前已开发到原 V1 阶段的一半,但有较多 bug、主链路最后一步还跑不通。
因此文档要收敛到一个我能真正做完、且能突出 Agent 能力的范围。

## 改写目标
1. 收敛范围,把重心从"把银行业务做全"转向"把 Agent 工程做深"。
2. 两份文档各自 5 分钟能读完。
3. 文档反映"实际会做的范围",不要写成 aspirational 愿景;指标只保留会真实测量的,
   并明确标注"目标"还是"实测";当前正在做的阶段如实标为进行中。

## 范围决策(改写的核心依据)

### 场景
- 只做一个场景:银企对账(企业账簿/ERP × 银行流水)。
- 删除第二场景"银行内部清算对账":架构里保留一句"可扩展到其他对账场景"的
  可扩展性说明即可,不要保留两套规则库/知识库/分支定义。

### Agent 核心能力(主角,要写深、写清设计决策)
- LangGraph 状态机 + 条件路由(显式建模出错/低置信/工具失败等非 happy path 分支)。
- 结构化输出可靠性:Pydantic Schema 校验 + 有界重试 + 兜底转人工。
- 多级 Fallback(三级即可,不要更复杂)。
- RAG:混合检索(Dense + BM25 + RRF)+ Reranker + 评测集(真实 Recall@5/MRR)。
- 可观测性:结构化日志 / 可回放 trace / token 成本统计 / Prompt 版本管理。
- 工具调用 + 只读/写权限边界(用普通函数工具,不用 MCP)。
- 确定性代码 vs LLM 的边界(金额计算、状态落库绝不交给 LLM,LLM 只读 READ-ONLY 结果)。
- Human-in-the-Loop 兜底。
- Agent 数量砍到 2 个:ExtractionAgent + AuditAgent(TraceAgent 标为可选)。
- "Hook 链与硬约束层":保留实质,但重新定位为"Agent 输出校验管线/护栏",
  只保留 4 个阶段:Schema 校验(+有界重试)→ 硬约束校验 → 决策/Fallback 路由 → 事务落库。
  删除吹捧性表述(不要写成"区别于普通 Demo 的核心架构创新")。
  鉴权/限流/缓存不再归入此管线:鉴权作为普通中间件,限流和缓存归入 LLM 客户端封装层。
  记忆相关的 hook(MemoryHook / MemoryUpdateHook)随记忆引擎一并删除。

### 后端(只保留"为 LLM 服务"的部分,删掉"跑大规模基础设施"的部分)
保留:
- FastAPI + Pydantic(骨架)。
- 异步任务队列 ARQ:上传即返回 task_id,Agent 在后台异步执行(LLM 慢,不能阻塞请求)。
- SSE:流式推送 Agent 执行步骤给前端。
- Redis:仅用于 LLM 结果缓存、API 调用限流、幂等去重(不用于记忆引擎)。
- MySQL + 事务:Agent 输出与台账原子落库(标准建表 + 合理索引)。
- 重试 + 熔断 + 优雅降级:围绕 LLM API 不稳定做可靠性处理。
- Docker Compose 一键启动。
- (可选)最简 JWT 登录。

### 删除 / 降级
- 三层记忆引擎 → 删,改为"AuditAgent 检索历史人工确认案例作为 few-shot"这一个轻量机制。
- 多租户 / 三级隔离架构 → 删;最多保留 JWT 带出的简单 user_id 行过滤。
- MCP 工具层 → 删。
- Nginx → 删(Docker 直接暴露端口)。
- 压测 / P50-P99 / 完整 SLA 指标表 / OWASP 清单 → 删(属于 SRE 信号,非本岗位关注)。
- MySQL 高级特性(HASH 分区 / 物化视图 / JSON 虚拟列索引)→ 删,降到标准水平。
- 云部署 → 降为"可选加分项",排在最后,不作为必做。

### 阶段重划(把原来的 MVP-0~V2 六个阶段压成三个)
- 阶段一(最小闭环):单场景银企对账;Pandas 清洗 + 确定性匹配 + YAML 规则;
  AuditAgent 真实 LLM 调用,结构化输出 + 校验 + 重试 + 兜底;基础 RAG;
  结构化日志能打出一条完整 trace。
- 阶段二(Agent 工程做深):LangGraph 状态机 + 条件路由;三级 Fallback;
  Prompt 独立文件 + 版本管理;RAG 评测集(真实 Recall@5/MRR)+ Agent 决策质量评测;
  工具调用权限边界;ExtractionAgent 接入。
- 阶段三(作品化):Vue 工作台 + SSE 展示 Agent 执行;ARQ 异步队列;
  Redis(缓存/限流/幂等);量化指标小面板;Docker Compose 一键起;
  (可选)JWT、云部署。
